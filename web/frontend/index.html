How It Fits

TelePath's core remains a C++ buffer engine. This browser layer provides a lightweight control surface for running workloads and inspecting recent results without changing the engine's execution path.

  • Data plane stays in the C++ engine.
  • Observation remains out of the hot path.
  • Run history and sweep results make trend inspection easier.
  • Future GUI and shared-memory telemetry can replace this shell without changing engine semantics.
